## Step 1 ‚Äî Run endpoint SQL

Single statement on `assets` PK (id), with `LEFT JOIN accounts` (issuer)
and `LEFT JOIN soroban_contracts` (contract). NULLs propagate cleanly
on irrelevant columns per the asset-identity CHECK contract. Runs on
all 3 picks.

The closed_at-vs-unix-timestamp equivalence is the protocol-truth
verification: stellar.expert's "created" field decodes to the exact
ledger-close timestamp our `ledgers` table records for sequence 62042818.
Confirms `soroban_contracts.deployed_at_ledger` is correctly threaded
into the assets endpoint response.

### Source B: Horizon

Skipped ‚Äî Horizon does not index Soroban contracts, so it cannot

## Step 5 ‚Äî Findings

### ‚úì Matches

- Shape covers every ¬ß6.9 DB column.
- LEFT JOINs correctly yield NULL on the irrelevant identity column
  per asset type (issuer NULL on native, contract NULL on classic
  credit if any existed).
- `token_asset_type_name(asset_type)` decodes to the expected
  display strings (`sac`, `native`).
- `soroban_contracts.deployed_at_ledger` is correctly surfaced on
  SACs and NULL on native ‚Äî matches the spec's "Soroban only" rule.
- Sample 1's `deployed_at_ledger = 62042818` matches stellar.expert's
  `created` timestamp byte-for-second.

### ‚öÝ Drift / known gaps (not bugs)

- **`name`, `icon_url`, `total_supply`, `holder_count` all NULL on
  the SAC samples** ‚Äî by spec. The SEP-1 enrichment layer + holder
  count tracking (task 0135) have not yet landed; the FE is
  expected to render gracefully ("‚Äî" / blank). No data error.
- **No classic_credit sample in this dataset.** Only 1 native + 7
  SACs in `assets` on the 30k window (see E08). E09 cannot exercise
  the classic-credit identity branch (issuer set, contract NULL)
  until the registry is fuller ‚Äî closes on full re-backfill /
  asset-extraction expansion.
- **S3 description / home_page not exercised here** ‚Äî 0164's S3
  layer is out of scope of the SQL-side audit and is documented as
  "missing object renders blank" per ¬ß6.9. No mismatch to flag.
